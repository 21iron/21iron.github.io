190429 Algorithm 01

[Test 002]

  • 키를 찾아라
    1. 현재 키가 있는 배열을 찾는다.
    2. 현재 키가 있는 배열 요소말고, 그 옆에 있는 요소가 키가 옮겨진 곳이다.
    3. 그 옆에 있는 요소를 새로운 배열에 넣는다.
    4. 그리고 다시 키가 있는 배열을 찾는다. 반복
  • 홀수 찾아라
    1. 2개의 For문을 이용하여 i값과 j값이 같은 곳을 찾는다.
    2. i값과 같은 j값을 새 배열에 넣는다.
    3. 그 배열의 길이가 짝수면 다 삭제하여 배열을 리셋
    4. 홀수가 나올 때 까지 반복, 만약 홀수가 나오면 리턴
    • 다른 사람들 코드를 보니까 count++이용하더라, 어느 배열의 길이를 알아보려고 할때 나도 count++를 사용해보도록 하자
  • 합후분가수찾기
    1. 2개의 For문, If절 이용
    2. If절에서 조건2가지를 추가, i값과 j값을 더한 값이 짝수 && i<j = true
    3. 위 2조건 부합하면, i,j값을 새 배열에 추가한다.
    4. 최종적으로 배열의 길이를 리턴한다.
    • 이것도 역시 count++를 이용하도록 하자
  • 우리집에 떨어진 사과와 오렌지를 찾아라
    1. 우리집의 범위 계산(for문 이용)
    2. 2개의 For문을 이용하여, 두개의 If절을 만든다.
    3. 1개의 If절은 사과가 떨어진 값 중에, 우리집 범위에 도달한 사과를 새 배열에 추가한다.
    4. 다른 1개의 If절은 위와 같이 진행하여 다른 새 배열에 추가한다.
    5. 사과와 오렌지 배열의 길이를 리턴한다.
    • 이것도 역시 ;; count++이용 하도록 하자
  • 교과서 적은 넘긴횟수를 찾아라
    1. 전체쪽수와 원하는페이지의 값을 이용하여, 첫 페이지부터 넘길 시의 공식을 구하자
    2. 전체쪽수와 원하는페이지의 값을 이용하여, 마지막 페이지부터 넘길 시의 공식을 구하자
    3. 두 값을 비교하여 적은 쪽을 리턴한다.
  • 느낀점?
    1. 다른 사람들의 알고리즘 해결 속도가 빠르다는걸 느꼈다.
    2. 컴퓨터가 아닌 나라면 어떻게 할까?의 생각으로 접근하자. 알고리즘 문제 받으면 무턱대고 어떤 코드쓰지로 다가가면 결국 내가 할줄아는 코드가 별로없어서 그걸 우려먹으려고 생각함. 물론 그것도 방식인데, 나라면 어떻게 풀까로 접근하여 하나하나 절차를 적으면서 해보자. 그리고 그걸 코딩으로 바꾸려고 해보자.
    3. 주어진 문제를 완벽히 이해하자. 몇번 문제들은 내가 문제를 제대로 이해하지 않고 시작하여서 하다가 다시 처음부터 한 적 도 있다. 특히 주어진 조건을 제대로 보지않고 하다가 문제를 다시보니 조건이 부합하지 않아서 계속 fail한 적도 있다. 문제를 꼼꼼히 보고, 조건을 제대로 인식하자. 머리가 나쁘면 적자 원철아.
    4. 본인의힘, 내 코드에 대한 이해, 알고리즘 결과의 정확성 너무 늦은 해결 속도에 대한 고민을 하자 ken님께서 주신 답변이다. 우선 3가지를 우선적으로 생각하라고 하셨으니, 우선 이 3가지를 우선적으로 고민하자.
